BEM

Block, Element, Modifier, it's a naming convention and methodology for writing CSS that makes your code more orgnized, scalable, and maintainable

Why use BEM

We always will have problems with CSS

  • Class Names can be confusing and unclear
  • Styles can accidentally affect unintented elements
  • Code becomes harder to maintain
  • Team Collaboration becomes difficult

BEM Solutions

  • Clear, descriptive and organized class names
  • Isolated components that do not interfere with each other
  • Easy to understand and maintanin code structure
  • Consistent naming across teams

BEM Naming Convention

block: The main component (header, menu, card)

block__element: Achild part of the block (menu__item, header__title, card__content, card__image, card__title)

block__element--modifier: A variation of the component or change state (button--small, button--large, menu--horizontal, menu--vertical, mode--light, mode--dark)
